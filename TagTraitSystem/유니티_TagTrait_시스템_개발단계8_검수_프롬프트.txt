유니티 TagTrait 시스템 개발 단계 8 검수 프롬프트

0. 검수 대상
- 개발 단계 8 - StackPolicy.Refresh
- Runtime/Core/TagInstance.cs

1. Refresh 적용 조건
- 기존 동일 Definition 소멸 태그인가
- StackPolicy.Refresh인가
- 기존 태그가 없으면 신규 추가 경로인가
- 영구 태그, None, 미구현 MaxDuration/StackCount는 false인가
- 같은 ID 다른 Definition은 Error와 false인가

2. 시간 교체 검수
- 새 정규화 duration으로 Duration과 RemainingTime을 모두 교체하는가
- 더 짧은 값도 무조건 적용하는가
- 더 긴 값도 적용하는가
- 현재 RemainingTime과 max를 사용하지 않는가
- 같은 값도 true인가
- 정규화는 호출 시 한 번만 수행하는가
- Tick 감소값을 재정규화하지 않는가

3. TagInstance 검수
- internal TryRefreshDuration
- 영구 상태에서는 false
- 소멸 상태에서 같은 값도 true
- Duration/RemainingTime 교체
- StackCount 유지
- Trait/이벤트/로그 없음
- 기존 Instance 참조 유지

4. Trait과 이벤트
- OnAdd/OnRemove/OnActivate 미호출
- Updated 한 번
- Reason.DurationRefreshed
- Previous 값 갱신 직전 상태
- Source 보존
- 콜백에서 새 상태 확인
- Added/Removed 없음
- 구독자 예외 후 상태 유지, 다음 구독자, true

5. 재진입
- processingTagIDs를 Updated 종료까지 유지
- 같은 ID 변경 차단
- 다른 ID 변경 허용
- 조회 허용
- 예외와 return 후 finally 해제

6. Tick 중 Refresh
- Refresh 성공을 우선하는가
- 성공한 Refresh를 무시하고 소멸을 우선하지 않는가
- private HashSet<TagID> tickModifiedTagIDs가 있는가
- isTicking 중 Refresh 성공 시 ID를 기록하는가
- 처리 전 기록된 ID를 건너뛰는가
- 다음 독립 Tick부터 감소하는가
- 먼저 감소된 뒤 Refresh되면 전체 새 시간이 최종값인가
- 제거 후 재추가는 새 Instance이며 다음 Tick부터 처리되는가
- 자기 만료 중 같은 ID Refresh는 재진입으로 차단되는가
- 강제 소멸 API를 조기 추가하지 않았는가

7. Tick 순회 및 정리
- 존재 여부, 참조 동일성, tickModified 여부, IsPerishable 순으로 재확인하는가
- 처리 순서와 무관하게 Refresh된 태그가 새 전체 시간으로 종료되는가
- Tick 시작과 finally에서 tickModifiedTagIDs를 Clear하는가
- isTicking도 finally에서 해제되는가
- 이전 Tick 기록이 다음 Tick에 남지 않는가

8. 실패 및 로그
- Definition/duration/Definition 불일치는 Error인가
- 영구/None/미구현 정책/재진입은 로그 없는 false인가
- 실패 시 Trait과 모든 변경 이벤트가 없는가
- 로그는 영어이며 단계 12 교체 주석이 있는가

9. 테스트 검수
- 기본 Refresh
- 짧은/긴 duration
- 최소 0.1과 midpoint
- 동일 Instance와 StackCount
- 같은 값 성공
- 이벤트 데이터와 Source
- Trait 미호출
- 구독자 예외
- 정책별 실패
- 같은 ID/다른 ID 재진입
- Tick 전/후 Refresh
- 제거 후 재추가
- 자기 만료 중 Refresh
- 순서 독립성
- tickModified 정리
- 다음 Tick 감소
- TearDown 정리

10. 기존 테스트 변경 검수
- 기존 소멸 중복 테스트를 정책별로 분리했는가
- Refresh 기대값만 의도적으로 true로 변경했는가
- None/MaxDuration/StackCount 임시 실패는 유지하는가
- 테스트 삭제나 무조건 완화가 없는가
- 단계 7 소멸→영구 전환 테스트가 유지되는가
