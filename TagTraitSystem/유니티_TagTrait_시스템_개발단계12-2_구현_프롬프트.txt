0. 문서 식별
- 개발 단계: 12-2
- 원래 개발 단계: 12

1. 현재 단계 목표
- Editor Assembly를 구성한다.
- 프로젝트 전체 TagDefinition 에셋을 수동으로 검사하는 Validator를 구현한다.
- 개별 Definition 오류, 중복 TagID, 중복 이름을 보고한다.
- Validator 핵심 로직과 AssetDatabase 검색을 분리해 테스트 가능하게 만든다.
- 문제를 자동 수정하지 않는다.
- 메뉴에서 사용자가 수동 실행할 수 있게 한다.

6. Validator 실행 방식
- 메뉴 경로:
-- Tools/TagTraitSystem/Validate Tag Definitions
- 수동 실행만 지원한다.
- 실행 순서:
-- AssetDatabase.FindAssets("t:TagDefinition")
-- GUID를 Asset Path로 변환
-- Asset Path를 StringComparer.Ordinal 기준 정렬
-- TagDefinition 로드
-- AssetRecord 목록 생성
-- 순수 검증 로직 호출
-- Issue 출력
-- 요약 출력
- Console을 자동으로 지우지 않는다.

8. 내부 결과 타입
- 다음 의미의 internal 타입을 구현한다.
-- internal enum TagValidationSeverity
--- Warning
--- Error
-- internal readonly struct TagDefinitionAssetRecord
--- TagDefinition Definition
--- string AssetPath
-- internal readonly struct TagValidationIssue
--- TagValidationSeverity Severity
--- TagDefinition Definition
--- string AssetPath
--- string Message
- 필드 또는 getter-only 프로퍼티를 사용한다.
- public API로 노출하지 않는다.
- Runtime Assembly에 두지 않는다.

9. 순수 검증 API
- 다음 의미의 internal static 메서드를 제공한다.
-- internal static IReadOnlyList<TagValidationIssue> Validate(IReadOnlyList<TagDefinitionAssetRecord> records)
- AssetDatabase를 호출하지 않는다.
- 입력 records를 변경하지 않는다.
- Definition 에셋 값을 변경하지 않는다.
- 결정적 결과 순서를 보장한다.
- 결과는 AssetPath Ordinal 순서와 검사 규칙 순서를 기반으로 정렬한다.

10. 개별 Definition Error
- Definition == null
-- Error
-- AssetPath 포함
- TagID == None
-- Error
- TagName == null 또는 string.Empty
-- Error
- Category == Keyword && Trait != null
-- Error
- Category != Keyword && Trait == null
-- Error
- StackPolicy == StackCount && MaxStackCount < 1
-- Error
- MaxStackCount == 1은 유효하다.
- StackCount가 아닌 정책의 MaxStackCount 값은 검사하지 않는다.

11. 이름 유효성
- 공백만 있는 TagName은 허용한다.
- 앞뒤 공백은 허용한다.
- 이름을 Trim하지 않는다.
- 대소문자를 변경하지 않는다.
- null과 string.Empty만 개별 Error로 처리한다.
- 공백 문자열의 중복은 Warning 대상이 될 수 있다.

12. 프로젝트 전체 중복 ID
- TagID.None은 중복 ID 그룹 검사에서 제외한다.
- 같은 non-None TagID를 사용하는 Definition이 둘 이상이면 Error이다.
- 충돌 에셋마다 한 Issue를 생성한다.
- 각 Issue 메시지에는:
-- TagID
-- 현재 AssetPath
-- 충돌한 다른 모든 AssetPath
를 포함한다.
- 현재 Definition을 Issue Context로 사용한다.
- 중복 그룹의 경로 목록은 Ordinal 정렬한다.
- 에셋 수를 자동으로 줄이거나 ID를 변경하지 않는다.

13. 프로젝트 전체 중복 이름
- null과 string.Empty는 중복 이름 검사에서 제외한다.
- 정확히 같은 문자열을 가진 Definition이 둘 이상이면 Warning이다.
- 비교:
-- StringComparer.Ordinal
-- 대소문자 구분
-- 앞뒤 공백 구분
- 공백만 있는 동일 문자열은 Warning 대상이다.
- 충돌 에셋마다 한 Issue를 생성한다.
- 메시지에 현재 경로와 다른 충돌 경로를 포함한다.

14. Issue 출력
- Error:
-- TagDiagnostics.LogError(issue.Message, issue.Definition)
- Warning:
-- TagDiagnostics.LogWarning(issue.Message, issue.Definition)
- TagDiagnostics가 접두어를 추가하므로 Validator 메시지에 접두어를 넣지 않는다.
- Warning은 Editor에서 출력된다.
- Context가 null이면 null 전달을 허용한다.

15. 요약 출력
- 모든 Issue 출력 후 한 번:
-- TagDiagnostics.Log("TagDefinition validation completed: X errors, Y warnings.")
- 문제가 없어도 0 errors, 0 warnings 요약을 출력한다.
- Issue 개수는 실제 생성한 Issue 수 기준이다.
- 충돌 그룹 수가 아니라 에셋별 Issue 수를 센다.

16. 자동 수정 금지
- ID 변경
- 이름 Trim
- Trait 제거 또는 생성
- MaxStackCount 변경
- 중복 에셋 삭제
- SerializedObject 변경
- SetDirty
- SaveAssets
- Undo 등록
- 에셋 Import
- Validator는 보고만 한다.

17. 결정적 순서
- AssetDatabase 검색 결과를 그대로 사용하지 않는다.
- AssetPath를 StringComparer.Ordinal로 정렬한다.
- 개별 Error는 고정된 검사 순서로 생성한다.
- 중복 ID 그룹은 TagID 숫자값 기준 정렬하거나 경로 기준으로 결정성을 확보한다.
- 중복 이름 그룹은 이름 Ordinal과 경로 Ordinal로 결정성을 확보한다.
- 테스트가 Dictionary 순회 순서에 의존하지 않도록 한다.

18. 로그 메시지
- 영어로 작성한다.
- 문제 유형, 값, AssetPath를 포함한다.
- 예:
-- TagDefinition at 'Assets/Tags/A.asset' uses TagID.None.
-- TagID 'TestAlpha' is duplicated by: Assets/Tags/A.asset, Assets/Tags/B.asset.
-- TagName 'Poison' is duplicated by: Assets/Tags/A.asset, Assets/Tags/C.asset.
- 긴 메시지라도 충돌 경로를 생략하지 않는다.

19. 테스트 데이터 생성
- 순수 Validate 테스트에서는 ScriptableObject.CreateInstance<TagDefinition>()을 사용한다.
- SerializedObject로 private 필드를 설정한다.
- 실제 프로젝트 에셋 파일을 만들지 않는다.
- TagDefinitionAssetRecord에 가상 AssetPath를 전달한다.
- TearDown에서 Object.DestroyImmediate로 정리한다.
- 메뉴/AssetDatabase 어댑터는 최소 정적 검토와 별도 단위 테스트로 검증한다.
