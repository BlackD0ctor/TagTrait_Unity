- Unity에서 실제 확인하지 않은 결과를 성공으로 보고하지 않았는지 확인한다.

3. 파일 및 네임스페이스 검수
- TagContainer 파일 경로가 정확한지 확인한다.
- 네임스페이스가 (TagTraitSystem.Runtime.Components)인지 확인한다.
- 테스트 네임스페이스가 (TagTraitSystem.Tests.EditMode)인지 확인한다.
- 동일 타입이 다른 경로나 네임스페이스에 중복 생성되지 않았는지 확인한다.
- Components 이외의 불필요한 Runtime 폴더가 조기 생성되지 않았는지 확인한다.

4. TagContainer 클래스 검수
- public class TagContainer : MonoBehaviour 형태인지 확인한다.
- sealed가 아닌지 확인한다.
- Awake, OnEnable, Start, Update가 없는지 확인한다.
- 영어 XML 문서 주석이 현재 책임만 설명하는지 확인한다.
- 태그 로직 외의 다른 컴포넌트 책임이 추가되지 않았는지 확인한다.

5. Dictionary 검수
- 필드가 다음 의미와 일치하는지 확인한다.
-- private readonly Dictionary<TagID, TagInstance>
-- 필드 선언 시 초기화
- SerializeField가 없는지 확인한다.
- Dictionary가 public, protected 또는 internal로 노출되지 않았는지 확인한다.
- Count 프로퍼티가 조기 추가되지 않았는지 확인한다.
- TagID.None을 Key로 넣는 경로가 차단되어 있는지 확인한다.
- 제거 API가 없는지 확인한다.

6. TryAddInstance 검수

6.1. 시그니처
- (internal bool TryAddInstance(TagInstance instance))인지 확인한다.
- public 또는 테스트 전용 이름이 아닌지 확인한다.

6.2. 입력 검증
- Null Instance에서 false인지 확인한다.
- Null Definition을 방어적으로 false 처리하는지 확인한다.
- TagID.None에서 false인지 확인한다.
- 상태 변경 없이 반환하는지 확인한다.

6.3. 정상 추가
- 기존 ID가 없으면 Dictionary에 추가하고 true인지 확인한다.
- Trait, 이벤트, StackPolicy 또는 시간 로직을 호출하지 않는지 확인한다.

6.4. 중복
- 같은 ID와 같은 Definition 참조이면 false, 로그 없음인지 확인한다.
- 같은 ID와 다른 Definition 참조이면 Error 로그 후 false인지 확인한다.
- 기존 Instance가 유지되는지 확인한다.
- 단계 12에서 TagDiagnostics로 교체한다는 한글 주석이 있는지 확인한다.
- 로그 문구가 영어인지 확인한다.

7. TagScan 검수
- 시그니처가 (public IReadOnlyCollection<TagInstance> TagScan())인지 확인한다.
- 빈 상태에서 Null이 아닌 빈 컬렉션인지 확인한다.
- Dictionary Values 실시간 뷰를 직접 반환하지 않는지 확인한다.
- 새 배열로 구성 스냅샷을 만드는지 확인한다.
- TagInstance를 깊은 복사하지 않는지 확인한다.
- 반환 순서에 의미를 부여하지 않는지 확인한다.
- LINQ ToArray를 불필요하게 사용하지 않았는지 확인한다.
- XML 문서 주석에 얕은 스냅샷과 순서 미보장이 설명되어 있는지 확인한다.

8. TagCheck 검수

8.1. Definition overload
- 시그니처가 정확한지 확인한다.
- Null이면 false인지 확인한다.
- None ID이면 false인지 확인한다.
- ID가 있으면 true인지 확인한다.
- Definition 참조 동일성을 요구하지 않는지 확인한다.
- 로그와 상태 변경이 없는지 확인한다.

8.2. ID overload
- 시그니처가 정확한지 확인한다.
- None이면 false인지 확인한다.
- Dictionary Key 존재 여부만 반환하는지 확인한다.
- 로그와 상태 변경이 없는지 확인한다.

9. TagANDCheck 검수
- 시그니처가 정확한지 확인한다.
- Null 배열과 빈 배열은 로그 없이 false인지 확인한다.
- 내부 Null과 None ID는 Error 로그 후 false인지 확인한다.
- 전체 배열을 먼저 검증한 뒤 보유 여부를 검사하는지 확인한다.
- 단일 입력이 TagCheck와 같은 결과인지 확인한다.
- 모든 ID가 있어야 true인지 확인한다.
- 중복 입력을 허용하고 결과에 영향이 없는지 확인한다.
- 불필요한 HashSet 할당이 없는지 확인한다.
- 단계 12 교체 주석과 영어 로그 문구를 확인한다.

10. TagORCheck 검수
- 시그니처가 정확한지 확인한다.
- Null 배열과 빈 배열은 로그 없이 false인지 확인한다.
- 내부 Null과 None ID는 Error 로그 후 false인지 확인한다.
- 전체 배열을 먼저 검증하는지 확인한다.
- 보유 태그가 앞에 있어도 뒤의 잘못된 입력을 놓치지 않는지 확인한다.
- 단일 입력이 TagCheck와 같은 결과인지 확인한다.
- 하나 이상 있으면 true인지 확인한다.
- 중복 입력을 허용하고 결과에 영향이 없는지 확인한다.
- 불필요한 HashSet 할당이 없는지 확인한다.
- 단계 12 교체 주석과 영어 로그 문구를 확인한다.

11. TryGetTag 검수
- 시그니처가 (public bool TryGetTag(TagDefinition definition, out TagInstance instance))인지 확인한다.
- 메서드 시작 시 out 값을 Null로 초기화하는지 확인한다.
- Null Definition이면 false와 Null인지 확인한다.
- None ID이면 false와 Null인지 확인한다.
- 같은 ID가 있으면 저장된 실제 Instance를 반환하는지 확인한다.
- 다른 Definition 참조라도 같은 ID이면 조회 성공하는지 확인한다.
- 없는 경우 false와 Null인지 확인한다.
- 로그와 상태 변경이 없는지 확인한다.
- TagID overload가 공개 API로 추가되지 않았는지 확인한다.
