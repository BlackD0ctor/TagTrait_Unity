4. Sample 전용 TagID
- 기존 값은 변경하지 않는다.
-- None = 0
-- TestAlpha = 1
-- TestBeta = 2
-- TestGamma = 3
- 다음 값을 추가한다.
-- SampleKeyword = 100
-- SamplePermanent = 101
-- SampleRefresh = 102
-- SampleMaxDuration = 103
-- SampleStackCount = 104
- 각 Sample enum 멤버에 영어 코드 주석을 작성한다.
- 주석은 해당 값이 Sample 전용이며 실제 프로젝트에서는 사용자가 자신의 ID 체계에 맞게 추가·교체할 수 있음을 알린다.
- 권장 주석 의미:
-- Sample-only TagID. Replace or extend it with project-specific IDs as needed.
- 기존 테스트용 ID 주석은 기존 계약을 유지한다.
- enum 숫자값은 명시적으로 작성한다.

5. Enum 테스트 수정
- EnumDefinitionTests에서 기존 값이 유지되는지 확인한다.
- 새 Sample ID 숫자값을 검증한다.
- 전체 enum 멤버 개수를 고정하는 테스트는 추가하지 않는다.
- 테스트 이름 예:
-- TagID_SampleValues_AreFixed
- 영어 주석 존재 여부를 Reflection이나 소스 문자열로 테스트하지 않는다.
- 주석은 정적 검수 항목으로 확인한다.

6. TagTraitSampleController 선언
- 경로:
-- Samples/Scripts/TagTraitSampleController.cs
- 네임스페이스:
-- TagTraitSystem.Samples
- 선언:
-- public sealed class TagTraitSampleController : MonoBehaviour
- 다음 특성을 사용한다.
-- [RequireComponent(typeof(TagContainer))]
- Runtime 기능을 재구현하지 않고 TagContainer 공개 API만 호출한다.

7. 직렬화 필드
- ([SerializeField] private TagDefinition selectedDefinition;)
- ([SerializeField] private float duration = 5f;)
- ([SerializeField] private bool autoTick = true;)
- 읽기 전용 공개 프로퍼티를 Sample 편의를 위해 임의로 추가하지 않는다.
- 테스트용 public setter를 추가하지 않는다.
- SerializedObject를 통해 테스트한다.

8. 내부 참조
- private TagContainer container;
- Awake에서 GetComponent<TagContainer>()로 캐시한다.
- RequireComponent를 사용하더라도 null 방어를 유지할 수 있다.
- Runtime 상태를 Sample Controller에 복제하지 않는다.

9. 자동 Tick
- Update에서 autoTick이 true일 때:
-- container.Tick(Time.deltaTime)
- TagContainer 자체에 Update를 추가하지 않는다.
- Sample은 외부 시간 관리자가 Tick을 호출해야 한다는 사용 예를 보여준다.
- autoTick false에서는 자동 감소가 없다.
- EditMode 테스트에서 Update 자동 실행을 검증하지 않는다.
- README에서 Play Mode 수동 확인 항목으로 안내한다.

10. ContextMenu 메서드
- 다음 public 또는 private ContextMenu 메서드를 제공한다.
-- AddPermanent()
-- AddPerishable()
-- Remove()
-- Activate()
-- TickOneSecond()
-- PrintCurrentTags()
- ContextMenu 표시 이름은 이해하기 쉬운 영어로 작성한다.
- 메서드는 selectedDefinition과 container를 사용한다.
- API source에는 gameObject를 전달한다.
- 반환값을 Sample 정보 로그로 출력할 수 있다.

11. null 설정 처리
- selectedDefinition == null이면:
-- TagDiagnostics.LogWarning
-- 상태 변경 없이 종료
- container == null이면:
-- TagDiagnostics.LogWarning
-- 종료
- Sample 설정 누락은 Error가 아니라 Warning이다.
- Warning은 Editor 또는 Development Build에서 출력된다.

21. Sample 테스트
- 기존 EditMode Test Assembly에서 작성한다.
- Test Assembly가 Sample Assembly를 참조한다.
- 실제 Scene이나 .asset을 만들지 않는다.
- GameObject에 Controller를 AddComponent하면 RequireComponent로 TagContainer가 존재하는지 확인한다.
- SerializedObject로 selectedDefinition, duration, autoTick을 설정한다.
- ContextMenu 메서드를 직접 호출할 수 있는 접근 수준이면 직접 호출한다.
- private일 경우 SendMessage나 Reflection 남용 대신 internal 테스트 가능 구조를 최소 범위로 검토한다.
- 테스트 전용 public API는 추가하지 않는다.

24. StackCount Tick 기술 부채
- README 주의사항에 다음을 간단히 기록한다.
-- Tick 콜백 중 다른 태그의 StackCount를 변경하면 현재 구조에서는 처리 순서에 따라 최종 StackCount가 달라질 수 있다.
-- 보류 기능 구현 또는 리팩터링 전에 StackCount Tick 처리 구조를 별도로 설계해야 한다.
- Sample 단계에서 해결하거나 숨기지 않는다.
