TagContainer Return Rules
- Normal no-op false: duplicate non-stack permanent add, max stack reached, lower MaxDuration input, activation of a missing tag, removing a missing tag, invalid query arrays.
- Error false with diagnostics: null definitions for mutation, TagID.None mutation, empty TagName for add/activate, invalid Keyword/Trait combination, missing Trait for non-keyword add/activate, same TagID with different definition, invalid duration, invalid StackCount state.
- Warning false: reentrant mutation for the same TagID.
- Nested Tick returns without throwing and logs a warning.

Duration and Tick
- PerishableTagAdd accepts positive finite duration only.
- Duration is rounded to one decimal place using MidpointRounding.AwayFromZero.
- Normalized duration is clamped to a minimum of 0.1.
- TimeEpsilon is 0.0001.
- Tick rejects negative, NaN, or infinite deltaTime with an error.
- Tick(0) is a no-op.

Trait and Event Order
- New non-keyword add: OnAdd, then OnTagAdded.
- Activation: OnActivate is called and its result is returned.
- Remove or expire: OnRemove, then OnTagRemoved.
- Refresh, MaxDuration, StackCount changes, and conversion: OnTagUpdated.
- Keyword tags do not call traits.

TagScan
- TagScan returns an array-backed snapshot collection.
- The snapshot composition does not update after later add/remove calls.
- TagInstance objects inside the snapshot are shared references.

Same ID Rules
- Same TagID and same TagDefinition means the operation can proceed according to policy.
- Same TagID and different TagDefinition is an error for mutation paths that detect it.
- Queries operate by TagID.
