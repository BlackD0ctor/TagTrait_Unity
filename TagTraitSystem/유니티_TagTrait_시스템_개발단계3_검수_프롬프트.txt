3. 공개 API 검수
- 정확한 시그니처:
-- public bool TagAdd(TagDefinition definition, GameObject source = null)
-- public bool TagSub(TagDefinition definition, GameObject source = null)
- source 이름과 기본값 확인
- 불필요한 overload 부재 확인
- source를 저장하거나 현재 상태 처리에 사용하지 않는지 확인

4. TagAdd 입력 검증

4.1. 차단
- Null Definition: Error + false
- TagID.None: Error + false
- Null/빈 TagName: Error + false
- Keyword + Trait: Error + false
- 비Keyword + Trait Null: Error + false
- 실패 시 Instance 생성 없음
- 실패 시 TryAddInstance 호출 없음
- 실패 시 상태 변경 없음

4.2. 허용
- Keyword + Trait Null 허용
- Category.None + Trait 존재 허용
- 공백만 있는 이름 허용
- 앞뒤 공백 이름 허용
- string.IsNullOrWhiteSpace로 차단하지 않음

5. TagAdd 정상 처리
- 유효성 검사 후 영구 TagInstance 생성
- TryAddInstance 재사용
- 별도 Dictionary 직접 추가 경로 없음
- 영구 초기값:
-- Duration 0f
-- RemainingTime 0f
-- StackCount 1
-- IsPerishable false
- Trait 호출 없음
- 이벤트 없음
- 성공 로그 없음

6. 중복 TagAdd 임시 정책
- 같은 Definition 중복이면 false
- 기존 Instance 유지
- StackPolicy와 관계없이 현재 단계에서는 실패
- StackCount 증가 없음
- Refresh/MaxDuration 조기 처리 없음
- 중복 분기 인접 한글 주석 확인:
-- 현재 단계에서는 모든 중복 영구 태그 추가를 실패 처리하며, StackCount 정책은 개발 단계 10에서 확장한다.
- 구현 결과 보고에 다음 수정 예정 단계 확인:
-- 단계 7: 소멸 태그의 영구 전환
-- 단계 10: 영구 StackCount 중복 추가
- 임시 테스트가 최종 정책으로 설명되지 않았는지 확인

7. 동일 ID 다른 Definition
- TagAdd에서 Error + false
- 기존 Instance 유지
- 새 Instance 미등록
- TagSub에서 Definition 참조 불일치 차단
- ID만 같다는 이유로 제거하지 않음

8. TagSub 입력 검증
- Null Definition: Error + false
- TagID.None: Error + false
- 다음을 제거 차단에 사용하지 않음:
-- 빈/공백 이름
-- Keyword-Trait 불일치
-- 비Keyword-Trait Null
-- Category.None

9. TagSub 정상 처리
- 같은 ID Instance 조회
- 기존 Instance.Definition과 요청 Definition 참조 비교
- 같은 참조일 때 완전 제거
- 제거 성공 true
- 제거 후 TagCheck false
- 제거 후 TryGetTag false와 Null
- Trait.OnRemove 없음
- 이벤트 없음
- StackCount 감소 없음
- public/internal 제거 helper 조기 추가 없음

10. TagSub 실패 처리
- 없는 태그: 로그 없이 false
- 같은 ID 다른 Definition: Error + false
- 실패 시 기존 상태 유지
- 다른 태그 영향 없음

11. 비정상 기존 데이터 제거
- internal TryAddInstance로 비정상 상태를 구성하는지 확인
- 다음을 TagSub로 제거 가능해야 함:
-- Keyword + Trait
-- 비Keyword + Trait Null
-- 빈 이름
- 공개 TagAdd로 비정상 데이터가 들어갔다고 가정하지 않음
- Trait 생명주기 호출 없음

12. 로그 검수
- Error 대상과 로그 없는 실패 구분
- 로그 문구 영어
- Debug.LogError 인접 단계 12 교체 주석
- 중복 TagAdd와 없는 TagSub에 Error 없음
- 성공 로그 없음
