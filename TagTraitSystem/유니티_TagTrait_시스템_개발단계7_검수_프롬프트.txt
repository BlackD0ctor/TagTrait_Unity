1. 검수 목적
- 소멸 -> 영구 단방향 전환이 정확히 구현되었는지 확인한다.
- 동일 TagInstance 참조와 StackCount가 보존되는지 확인한다.
- Trait 생명주기가 재호출되지 않는지 확인한다.
- ChangedToPermanent Updated 이벤트 데이터와 발생 시점을 확인한다.
- 기존 Tick, 재진입, 중복 정책의 회귀 여부를 확인한다.
- 단계 8~11 기능이 조기 구현되지 않았는지 확인한다.

3. TagInstance 전환 API
- internal bool TryConvertToPermanent 시그니처 확인
- 이미 영구이면 false인지 확인
- 소멸 상태이면 true인지 확인
- Duration = 0f
- RemainingTime = 0f
- IsPerishable = false
- StackCount 유지
- Definition 참조 유지
- Trait, 이벤트, 로그, Dictionary 접근 없음
- setter 또는 메서드가 불필요하게 public으로 노출되지 않았는지 확인

4. 동일 Instance 유지
- 전환 전후 참조 동일성 확인
- Dictionary 값 교체 없음
- 새 TagInstance 생성 없음
- 기존 TagScan 얕은 스냅샷에서 전환 상태 관찰 가능
- 외부 참조 단절 없음

5. 전환 방향
- 소멸 태그 + TagAdd는 true와 영구 전환
- 영구 태그 + PerishableTagAdd는 false
- 영구 -> 소멸 기능이 없는지 확인
- RemainingTime 크기를 전환 제한 조건으로 사용하지 않는지 확인

6. StackCount 검수
- 전환 시 기존 StackCount 보존
- 1로 강제 초기화하지 않음
- 증가 또는 감소 없음
- ChangedToPermanent와 StackIncreased를 같은 호출에서 함께 처리하지 않음
- 단계 10·11 인계가 보고에 명시됨

7. TagAdd 분기 검수
- Definition 입력 검증 유지
- 동일 ID 가드 획득 유지
- 기존 Instance 조회 후 Definition 참조 비교
- 다른 Definition이면 Error + false
- 같은 Definition 소멸이면 전환
- 같은 Definition 영구이면 기존 중복 false
- 소멸 전환 경로에서 TryAddInstance를 호출하거나 새 객체를 만들지 않음

8. Trait 생명주기 검수
- 전환 시 OnAdd 미호출
- OnRemove 미호출
- OnActivate 미호출
- 정상 Keyword에서도 Trait 미호출
- 기존 Trait 호출 횟수 유지

9. Updated 이벤트 검수
- 성공 전환당 정확히 한 번
- Reason == ChangedToPermanent
- Container 정확성
- Definition은 기존 Instance.Definition
- Instance는 동일 전환 객체
- Source는 TagAdd 전달값
- PreviousDuration은 전환 직전 값
- PreviousRemainingTime은 전환 직전 값
- PreviousStackCount는 전환 직전 값
- 콜백 시 현재 상태가 영구 상태
- Added와 Removed 미발생

10. 이벤트 예외 검수
- SafeInvoke 사용
- 첫 구독자 예외 후 다음 구독자 호출
- 상태 롤백 없음
- TagAdd true 유지
- 예외 비전파
- Error 로그와 단계 12 교체 주석 유지
- 예외 이후 재진입 가드 정상 해제

11. 재진입 검수
- Updated 이벤트 종료까지 processingTagIDs 유지
- 같은 ID TagAdd 차단
- 같은 ID TagSub 차단
- 같은 ID PerishableTagAdd 차단
- 같은 ID TagActivate 차단
- 추가 상태 변화와 이벤트 없음
- 다른 ID 변경 허용
- 조회 API 허용
- finally에서 가드 해제

12. Tick 연동 검수
- 전환 후 Tick에서 감소하지 않음
- Tick에서 IsPerishable 재검사 유지
- Tick 스냅샷에 포함되었더라도 처리 전 전환되면 건너뜀
- 전환 Instance를 만료시키지 않음
- Tick 구조와 isTicking 정책의 불필요한 변경 없음

13. PerishableTagAdd 임시 정책 검수
- 기존 영구 태그에 false
- 기존 소멸 태그에 false
- 시간 갱신 없음
- 전환 없음
- 스택 변화 없음
- Trait과 이벤트 없음
- 같은 ID 다른 Definition은 Error + false
- 단계 8, 9, 11 인계 명시

14. 기존 테스트 정책 변경 검수
- 영구 태그 중복 TagAdd false 테스트 유지
- 기존 소멸 태그 + TagAdd false 테스트가 있었다면 전환 성공 기대값으로 의도적으로 수정
- 테스트를 삭제하거나 임의 완화하지 않음
- 변경 이유가 보고됨
- 단계 6 PerishableTagAdd 중복 false 테스트 유지

15. 필수 테스트 검수 - 상태
- 전환 true
- 동일 Instance
- Duration/RemainingTime 0
- IsPerishable false
- StackCount 보존
- Tick 무영향
- 기존 TagScan 스냅샷 반영

16. 필수 테스트 검수 - Trait과 이벤트
- Trait 세 콜백 미호출
- Keyword 미호출
- Updated 한 번
- ChangedToPermanent
- Previous 값
- Source
- 최종 상태 조회
- Added/Removed 미발생
- 구독자 예외 정책

17. 필수 테스트 검수 - 실패와 재진입
- 기존 영구 TagAdd false
- 영구 StackCount 정책도 단계 10 전 false
- 영구/소멸 기존 태그 PerishableTagAdd false
- 다른 Definition 오류
- 실패 이벤트 없음
- 동일 ID 재진입 네 API 차단
- 다른 ID 허용
- 예외 후 가드 해제
- Tick 처리 전 전환된 스냅샷 항목 건너뜀

18. 기존 기능 회귀
- 신규 영구 TagAdd의 Added 이벤트 유지
- 신규 소멸 PerishableTagAdd의 Added 이벤트 유지
- TagSub와 만료 Removed 이벤트 유지
- TagActivate 정책 유지
- Tick 입력과 만료 정책 유지
- Definition 유효성 검사 유지
