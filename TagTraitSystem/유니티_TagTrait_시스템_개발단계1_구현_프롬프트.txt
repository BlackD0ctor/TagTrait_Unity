1. AI 역할
- Unity 6 환경의 Runtime 데이터 모델과 EditMode 테스트를 작성하는 Unity C# 개발자로 작업한다.
- 현재 단계에서는 TagTrait 시스템의 enum, 원본 태그 정의, 런타임 태그 상태 및 최소 Trait 기반 타입만 구현한다.
- 태그 조회, 추가, 제거, Trait 실행, 이벤트 및 시간 처리 로직은 구현하지 않는다.
- Unity Editor 내부 컴파일과 Test Runner 실행은 수행했다고 간주하지 않는다.
- 실제 컴파일과 테스트 실행은 사용자가 Unity에서 수행한다.

5. AI 작업 완료 기준
- 지정된 Runtime 파일이 생성 또는 수정되어 있다.
- 지정된 EditMode 테스트 파일이 생성되어 있다.
- 각 enum 숫자값이 명시적으로 고정되어 있다.
- TagDefinition의 직렬화 필드와 읽기 전용 프로퍼티가 요구사항과 일치한다.
- MaxStackCount의 기본값이 1이며 TagDefinition에만 저장된다.
- TagInstance의 두 생성자와 초기 상태가 요구사항과 일치한다.
- TagTraitSO가 단계 1 범위의 최소 추상 타입으로 구현되어 있다.
- 테스트용 Unity Object가 TearDown에서 정리되도록 작성되어 있다.
- 실제 Unity 컴파일과 테스트는 미실행 상태로 명확히 보고한다.

8. Enum 구현

-- (TestAlpha = 1)

-- (TestGamma = 3)

10. TagDefinition 구현

10.1. 클래스 선언
- ScriptableObject를 상속하는 public sealed class로 구현한다.
- CreateAssetMenu를 다음 값으로 설정한다.
-- fileName: (NewTagDefinition)
-- menuName: (TagTraitSystem/Tag Definition)
-- order: (0)

10.2. 직렬화 필드
- 다음 필드를 정확한 이름과 형식으로 구현한다.
-- ([SerializeField] private TagID tagID;)
-- ([SerializeField] private string tagName = string.Empty;)
-- ([SerializeField] private TagCategory category;)
-- ([SerializeField] private TagTraitSO trait;)
-- ([SerializeField] private StackPolicy stackPolicy;)
-- ([SerializeField] private int maxStackCount = 1;)
-- ([SerializeField] private ExclusiveGroup exclusiveGroup;)
-- ([SerializeField] private int priority;)
-- ([SerializeField] private bool isSaveable;)
- private 필드는 별도 접두어 없이 camelCase를 사용한다.
- 불필요한 public setter를 제공하지 않는다.
- Unity 직렬화에 필요하지 않은 추가 필드를 만들지 않는다.

10.3. 읽기 전용 프로퍼티
- 다음 프로퍼티를 정확한 이름으로 구현한다.
-- (public TagID TagID => tagID;)
-- (public string TagName => tagName;)
-- (public TagCategory Category => category;)
-- (public TagTraitSO Trait => trait;)
-- (public StackPolicy StackPolicy => stackPolicy;)
-- (public int MaxStackCount => maxStackCount;)
-- (public ExclusiveGroup ExclusiveGroup => exclusiveGroup;)
-- (public int Priority => priority;)
-- (public bool IsSaveable => isSaveable;)
-- (public bool IsKeywordTag => category == TagCategory.Keyword;)

10.4. 책임
- TagDefinition은 원본 설정 데이터만 저장한다.
- MaxStackCount는 태그별 최대 스택 설정의 원본이며 기본값은 1이다.
- MaxStackCount는 StackPolicy.StackCount에서만 Runtime 동작에 사용하고 다른 정책에서는 설정값만 보존한다.
- MaxStackCount == 1은 유효한 설정이다.
- TagInstance에 최대 스택 수를 별도 필드로 복제하지 않는다.
- 오브젝트별 최대치 또는 Runtime override 구조를 구현하지 않는다.
- 런타임 시간, 남은 시간, 현재 스택 수 및 유한성 상태를 저장하지 않는다.
- 여러 오브젝트가 같은 TagDefinition을 공유할 수 있어야 한다.
- IsKeywordTag는 Category만 확인한다.
- Trait Null 여부를 IsKeywordTag 판정에 사용하지 않는다.

10.5. 현재 단계에서 구현하지 않을 검증
- public IsValid 프로퍼티를 추가하지 않는다.
- public Validate 메서드를 추가하지 않는다.
- OnValidate를 구현하지 않는다.
- 프로젝트 전체 중복 ID 검색을 구현하지 않는다.
- 빈 이름, Keyword-Trait 조합 및 비Keyword-Trait Null을 현재 단계 Runtime 코드에서 차단하지 않는다.
- MaxStackCount가 1 미만인지 현재 단계 Runtime 코드에서 차단하거나 보정하지 않는다.
- MaxStackCount == 1을 오류로 처리하지 않는다.
- 위 데이터 유효성 처리는 이후 API 및 Editor Validator 단계에서 구현한다.

10.6. 문서화
- public 클래스와 public 프로퍼티에 필요한 영어 XML 문서 주석을 작성한다.
- 일반 구현 설명이 필요한 경우 한글 코드 주석을 사용할 수 있다.
- 보류 필드인 ExclusiveGroup, Priority, IsSaveable이 현재 Runtime 동작에 사용되지 않는다는 점을 과도한 주석으로 반복하지 않는다.

11. TagInstance 구현

11.1. 클래스 선언
- 다음 형태로 구현한다.
-- (public sealed class TagInstance)

11.2. 프로퍼티
- 다음 프로퍼티를 구현한다.
-- (public TagDefinition Definition { get; })
-- (public float Duration { get; private set; })
-- (public float RemainingTime { get; private set; })
-- (public int StackCount { get; private set; })
-- (public bool IsPerishable { get; private set; })
- 외부에서 상태를 임의로 변경할 수 있는 public setter를 제공하지 않는다.
- 현재 단계에서는 상태 변경용 public 또는 internal 메서드를 추가하지 않는다.

11.3. 영구 태그 생성자
- 다음 시그니처로 구현한다.
-- (internal TagInstance(TagDefinition definition))
- definition이 Null이면 (ArgumentNullException)을 발생시킨다.
- 정상 생성 시 초기값은 다음과 같다.
-- Definition = 전달된 definition
-- Duration = 0f
-- RemainingTime = 0f
-- StackCount = 1
-- IsPerishable = false

11.4. 소멸 태그 생성자
- 다음 시그니처로 구현한다.
-- (internal TagInstance(TagDefinition definition, float duration))
- definition이 Null이면 (ArgumentNullException)을 발생시킨다.
- 정상 생성 시 초기값은 다음과 같다.
-- Definition = 전달된 definition
-- Duration = 전달된 duration
-- RemainingTime = 전달된 duration
-- StackCount = 1
-- IsPerishable = true

11.5. 시간값 책임
- 단계 1에서는 duration을 정규화하지 않는다.
- 반올림, 최소 0.1초 보정, NaN 및 Infinity 검증을 구현하지 않는다.
- TagInstance는 이미 검증 및 정규화되었다고 가정한 값을 저장하는 데이터 객체로 둔다.
- 시간 정규화는 개발 단계 6에서 구현한다.

11.6. 문서화
- 클래스와 public 프로퍼티에 영어 XML 문서 주석을 작성한다.
- 생성자가 internal인 이유를 필요 시 한글 주석으로 간단히 설명할 수 있다.
- TagContainer가 아직 존재하지 않으므로 미래 기능을 가정한 상태 변경 메서드를 추가하지 않는다.

16. TagDefinitionTests

16.1. 테스트 객체 생성
- 각 테스트에서 (ScriptableObject.CreateInstance<TagDefinition>())를 사용한다.
- Trait 테스트가 필요하면 (ScriptableObject.CreateInstance<TestTagTraitSO>())를 사용한다.
- 생성한 Unity Object는 필드에 보관한다.
- ([TearDown])에서 (Object.DestroyImmediate)를 사용하여 정리한다.

16.2. private 직렬화 필드 설정
- UnityEditor의 (SerializedObject)와 (SerializedProperty)를 사용한다.
- Runtime에 테스트용 setter, 생성자 또는 초기화 API를 추가하지 않는다.

21. 금지 사항
- 단계 2 이후의 기능을 미리 구현하지 않는다.
- TagDefinition에 현재 StackCount 같은 런타임 상태를 저장하지 않는다.

- TagInstance에 MaxStackCount를 별도 필드나 복제 상태로 추가하지 않는다.
- 오브젝트별 최대치 또는 Runtime override 기능을 추가하지 않는다.
- TagTraitSO에 오브젝트별 런타임 상태를 저장하지 않는다.
- TagInstance 상태에 public setter를 추가하지 않는다.
- TagInstance 생성자를 public으로 변경하지 않는다.
- Runtime에 테스트 전용 public API를 추가하지 않는다.
- IsValid 또는 Validate 공개 API를 임의로 추가하지 않는다.
- 시간값을 단계 1에서 정규화하지 않는다.
- Keyword 여부를 Trait Null로 판정하지 않는다.
- enum 값을 암시적 순서에 맡기지 않고 숫자값을 명시한다.
- 기존 테스트를 삭제하거나 기대값을 완화하지 않는다.
- 실제 테스트를 실행하지 않고 통과했다고 보고하지 않는다.
- Unity 컴파일을 확인하지 않고 오류가 없다고 단정하지 않는다.
