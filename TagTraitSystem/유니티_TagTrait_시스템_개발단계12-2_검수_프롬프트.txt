유니티 TagTrait 시스템 개발 단계 12-2 검수 프롬프트

- 개발 단계: 12-2
- 단계 제목: Editor TagDefinition Validator
- 프롬프트 종류: 검수 프롬프트
- 기준 문서:
-- 수정된 AI 개발 공통 규칙

-- 개발 단계 12-2 구현 결과 보고

1. 검수 목적
- Editor Assembly와 참조 방향을 확인한다.
- 순수 Validator와 AssetDatabase 어댑터 분리를 확인한다.
- 개별 오류, 중복 ID, 중복 이름 규칙을 확인한다.
- 결정적 결과와 자동 수정 금지를 확인한다.
- Runtime 코드에 UnityEditor 의존성이 없는지 확인한다.

2. 검수 대상
- Editor/TagTraitSystem.Editor.asmdef
- Editor/AssemblyInfo.cs
- Editor/Validation/TagDefinitionValidator.cs
- Editor/Validation/TagDefinitionValidationMenu.cs
- Tests/EditMode/TagDefinitionValidatorTests.cs
- Tests/EditMode asmdef 조건부 수정
- 구현 결과 보고

3. Editor Assembly 검수
- 이름이 TagTraitSystem.Editor인가
- Runtime만 참조하는가
- includePlatforms가 Editor인가
- Runtime이 Editor를 참조하지 않는가
- Test Assembly가 Editor를 참조하는가
- 불필요한 Assembly 변경이 없는가

4. InternalsVisibleTo 검수
- TagTraitSystem.Tests.EditMode만 허용하는가
- Assembly 이름이 정확한가
- public key를 임의로 추가하지 않았는가
- internal Validator 테스트가 가능한가

5. 실행 방식 검수
- 메뉴 경로가 Tools/TagTraitSystem/Validate Tag Definitions인가
- 수동 실행만 지원하는가
- OnValidate/AssetPostprocessor/빌드 전 자동 실행이 없는가
- Console을 지우지 않는가

6. AssetDatabase 어댑터 검수
- FindAssets("t:TagDefinition") 사용
- GUID→Path
- Ordinal Path 정렬
- Definition 로드
- Record 생성
- 순수 Validate 호출
- 로그 및 요약 출력
- 핵심 검증 로직이 AssetDatabase에 결합되지 않았는가

7. 결과 타입 검수
- internal enum Severity
- internal readonly struct Record
- internal readonly struct Issue
- getter-only 또는 readonly 구조
- public API로 노출하지 않았는가
- Runtime에 위치하지 않는가

8. 순수 Validate 검수
- IReadOnlyList<Record> 입력인가
- IReadOnlyList<Issue> 출력인가
- AssetDatabase 호출이 없는가
- 입력과 에셋을 변경하지 않는가
- 결과 순서가 결정적인가

9. 개별 Error 검수
- null Definition
- TagID.None
- null/empty 이름
- Keyword + Trait
- 비Keyword + null Trait
- StackCount + MaxStackCount < 1
- MaxStackCount 1 허용
- 다른 정책의 MaxStackCount 미검사
- 공백 이름 허용
- 앞뒤 공백 허용

10. 중복 ID 검수
- non-None만 검사
- 에셋마다 한 Issue
- 모든 충돌 경로 포함
- 현재 Definition Context
- 경로 정렬
- 자동 ID 변경 없음

11. 중복 이름 검수
- null/empty 제외
- StringComparer.Ordinal
- 대소문자 구분
- 앞뒤 공백 구분
- 공백 동일 문자열은 Warning
- 에셋마다 한 Issue
- 모든 충돌 경로 포함

12. 로그 검수
- Error는 TagDiagnostics.LogError
- Warning은 TagDiagnostics.LogWarning
- 요약은 TagDiagnostics.Log
- Validator 메시지에 접두어 중복 없음
- 영어 메시지
- AssetPath와 값 포함
- 0/0 요약도 출력

13. Issue 개수 검수
- 실제 에셋별 Issue 수 기준인가
- 그룹 수로 세지 않는가
- 세 에셋 중복이면 세 Issue인가
- 요약 개수와 출력 Issue 수가 일치하는가

14. 자동 수정 금지 검수
- ID 변경 없음
- 이름 Trim 없음
- Trait 변경 없음
- MaxStackCount 변경 없음
- SetDirty 없음
- SaveAssets 없음
- Undo 없음
- 에셋 삭제 없음
- Import 없음

15. 결정적 순서 검수
- 입력 순서와 무관한가
- Path Ordinal 정렬인가
- 중복 그룹과 Issue 순서가 안정적인가
- 테스트가 Dictionary 순회 순서에 의존하지 않는가

16. 테스트 검수
- 개별 Error
- 허용 이름
- invalid maximum
- 중복 ID
- 중복 이름
- 대소문자/공백 비교
- per-asset Issue
- 무변경
- 결정적 순서
- 요약
- TearDown
- 실제 프로젝트 에셋 미생성

17. Runtime/Editor 경계 검수
- Runtime 코드에 UnityEditor using이 없는가
- Runtime asmdef가 Editor를 참조하지 않는가
- Editor 타입이 Runtime namespace에 있지 않은가
- 메뉴와 Validator가 Editor Assembly에만 있는가

18. 범위 초과 검수
- 자동 수정
- 자동 실행
- Custom Inspector

- 마이그레이션

- StackCount Tick 재설계
- ExclusiveGroup/Priority/IsSaveable 기능
- 조기 구현 시 영향 범위 보고

19. 차단 문제
