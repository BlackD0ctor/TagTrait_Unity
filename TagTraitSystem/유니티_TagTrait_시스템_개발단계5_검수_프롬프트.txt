- TagActivate 입력 검증, Context, 반환값과 이벤트 부재를 확인한다.
- Trait 예외 격리와 재진입 가드 유지 범위를 확인한다.
- Keyword와 비정상 데이터 복구 정책을 확인한다.
- 단계 6 이후 기능 조기 구현 여부를 확인한다.
- 미실행 결과의 허위 보고 여부를 확인한다.

3. TraitContext 검수
- public readonly struct인지 확인
- 네임스페이스 확인
- getter-only 프로퍼티 확인:
-- Container
-- Target
-- Source
-- Definition
-- Instance
- internal 생성자 확인
- public setter와 public 생성자 부재 확인
- Target이 container.gameObject인지 확인
- Source를 Target 대체값으로 사용하지 않는지 확인
- 전달값 보존 확인
- XML 문서 주석 영어 확인

4. ITagTrait 검수
- public interface인지 확인
- 정확한 메서드 확인:
-- void OnAdd(TraitContext context)
-- void OnRemove(TraitContext context)
-- bool OnActivate(TraitContext context)
- ref/in/out, async, IEnumerator 부재 확인

5. TagTraitSO 검수
- ScriptableObject와 ITagTrait 구현 확인
- OnAdd public virtual void, 빈 기본 구현
- OnRemove public virtual void, 빈 기본 구현
- OnActivate public abstract bool
- CreateAssetMenu 부재
- Runtime 상태 필드 부재
- 구체 오브젝트 참조 저장 부재

6. TestTagTraitSO 검수
- 기존 테스트 호환을 위한 concrete class 유지
- OnActivate만 최소 구현
- 기본 true 반환
- Recording 기능이나 상태를 불필요하게 추가하지 않음

7. RecordingTagTraitSO 검수
- Test Assembly에만 존재
- 호출 횟수와 마지막 Context 기록 가능
- ActivateResult 설정 가능
- 콜백 Action 및 예외 설정 가능
- CreateAssetMenu 없음
- Runtime 공개 계약 오염 없음

8. TagAdd 생명주기 검수
- 단계 3 입력 검증 유지
- 동일 ID 가드 획득 후 처리
- Dictionary 등록 후 OnAdd 호출
- OnAdd 후 OnTagAdded 발생
- OnAdd 시 조회에서 태그가 존재
- Keyword 정상 추가 시 OnAdd 없음, Added 이벤트는 있음
- 실패 Add에서 OnAdd와 이벤트 없음
- 동일 Definition 중복 임시 정책 유지
- OnAdd 예외 후 상태 유지, Added 이벤트, true
- OnAdd 예외 비전파

9. TagSub 생명주기 검수
- 단계 3 입력 검증 유지
- 동일 ID 가드 획득 후 처리
- OnRemove를 Dictionary 제거 전에 호출
- OnRemove 시 조회에서 태그가 존재
- OnRemove 후 제거
- 제거 후 OnTagRemoved 발생
- 실패 Sub에서 OnRemove와 이벤트 없음
- OnRemove 예외 후 제거 계속, Removed 이벤트, true
- OnRemove 예외 비전파

10. 비정상 데이터 제거 검수
- Keyword + Trait 기존 데이터 제거 허용
- 해당 Trait.OnRemove 미호출
- 별도 Trait 불일치 Error 로그 없음
- 비Keyword + Trait Null 제거 허용
- OnRemove 없음
- 별도 Trait 불일치 Error 로그 없음
- OnTagRemoved 발생
- 빈 이름 기존 데이터 제거 정책 유지

11. Trait 예외 안전성 검수
- OnAdd 예외: Error, 상태 유지, Added 이벤트, true
- OnRemove 예외: Error, 제거, Removed 이벤트, true
- OnActivate 예외: Error, false, 상태 유지, 이벤트 없음
- 예외 재전파 없음
- 단계 12 교체 한글 주석
- 로그 문구 영어
- 예외 이후 가드 해제 및 후속 API 가능

12. TagActivate 공개 API 검수
- 정확한 시그니처:
-- public bool TagActivate(TagDefinition definition, GameObject source = null)
- 불필요 overload 부재
- source 저장 부재

13. TagActivate 입력 검증 검수
- Null Definition: Error + false
- TagID.None: Error + false
- Null/빈 이름: Error + false
- 정상 Keyword + Null Trait: 로그 없이 false
- Keyword + Trait: Error + false, Trait 미호출
- 비Keyword + Trait Null: Error + false
- 미보유 정상 태그: 로그 없이 false
- 같은 ID 다른 Definition: Error + false
- 실패 시 상태 변경과 이벤트 없음

14. TagActivate 정상 실행 검수
- 보유 중인 정상 비Keyword만 실행
- 실제 Instance.Definition 사용
- Context.Container 정확
- Context.Target == container.gameObject
- Context.Source 전달값 보존
- Context.Instance 실제 참조
- OnActivate 반환값 그대로 반환
- true 강제 없음
- TagActivate 자체 변경 이벤트 없음

15. Category 정책 검수
- Keyword 여부는 Category만 기준
- 정상 Keyword Trait 없음
- Keyword Trait 실행 금지
- Category.None + Trait은 일반 태그로 OnAdd/OnRemove/OnActivate 호출 가능
- 비Keyword Trait Null 정상 추가 금지 유지

16. 재진입 가드 검수
- Trait 콜백과 뒤따르는 이벤트 종료까지 같은 ID 유지
- try-finally 해제
- OnAdd 같은 ID Add/Sub 차단
- OnRemove 같은 ID Add/Sub 차단
- OnActivate 같은 ID Add/Sub/Activate 차단
- 상태 변경과 추가 이벤트 없음
- 다른 ID Add/Sub/Activate 허용
- 조회 API 허용
- 교차 재진입 시 원래 ID 차단
- 자기 TagSub 우회 기능 없음

17. TraitContextTests 검수
