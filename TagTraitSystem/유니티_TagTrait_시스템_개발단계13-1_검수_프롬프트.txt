1. 검수 목적
- 여러 단계의 공개 API가 대표 사용 흐름에서 연결되는지 확인한다.
- 통합 테스트가 단위 테스트를 과도하게 반복하지 않는지 확인한다.
- Runtime 기능이나 공개 API가 새로 추가되지 않았는지 확인한다.
- PlayMode Assembly가 조기 추가되지 않았는지 확인한다.

2. 검수 대상
- Tests/EditMode/TagTraitSystemIntegrationTests.cs
- 조건부 IntegrationRecordingTraitSO.cs
- 단계 13-1에서 수정된 테스트 파일
- 구현 결과 보고

3. 테스트 형식 검수
- 기존 EditMode Assembly인가
- PlayMode Assembly가 없는가
- Tick을 직접 호출하는가
- Scene/Coroutine/물리에 의존하지 않는가
- Unity Object TearDown이 있는가

4. 통합 Trait 검수
- Test Assembly에만 존재하는가
- 필요한 기록만 보유하는가
- Runtime에 추가되지 않았는가
- 테스트 전용임을 명확히 하는가
- DestroyImmediate로 정리되는가

5. 영구 생명주기 검수
- Add→OnAdd→Added
- Activate→OnActivate
- Sub→OnRemove→Removed
- TraitContext Target/Source/Container/Definition/Instance
- 같은 Instance
- 최종 미보유

6. Keyword 검수
- Add/Removed 상태 정상
- Activate false
- Trait 없음
- 로그 없음
- IsKeywordTag 계약 유지

7. Refresh 검수
- 최초 5초
- Tick 후 3초로 Refresh
- Duration/RemainingTime 3
- DurationRefreshed
- OnAdd 중복 없음
- Expired

8. MaxDuration 검수
- 실제 연장만 성공
- DurationExtendedToMax
- 짧은 입력 no-op
- 무로그/무이벤트
- 최종 만료

9. StackCount 검수
- 같은 Duration
- 추가 시 RemainingTime 초기화
- 일부 소진 StackDecreased
- 큰 Tick 초과시간 전달
- 중간 Trait 생명주기 없음
- StackCount 0 미사용
- 최종 제거 계약

10. 전환 검수
- 같은 Instance
- 시간 0
- StackCount 보존
- ChangedToPermanent
- 같은 호출 증가 없음
- 이후 Tick 무영향

11. 여러 ID 독립성 검수
- 정책별 태그 동시 보유
- 한 ID 변경이 다른 ID에 영향 없음
- 개별 만료
- TagCheck/TryGetTag/TagScan 일관성
- TagScan 얕은 스냅샷 의미

12. Diagnostics 검수
- Error
- Warning
- silent false
- 대표 흐름만 확인
- 모든 단위 로그 테스트를 중복하지 않는가

13. 이벤트 순서 검수
- 최초 추가
- 마지막 제거
- 만료
- 중간 StackCount
- 같은 ID 이벤트 최대 한 번

14. 범위 초과 검수
- Runtime API 추가
- PlayMode Assembly
- Sample 코드
- Scene/Asset 생성
- StackCount Tick 재설계
- 성능 벤치마크
- 조기 구현 시 영향 범위 보고

15. 차단 문제
- Runtime 공개 API 변경
- 단위 테스트 삭제
- PlayMode Assembly 추가
- Trait/Event 순서 기대 오류
- StackCount 시간 초기화 누락
