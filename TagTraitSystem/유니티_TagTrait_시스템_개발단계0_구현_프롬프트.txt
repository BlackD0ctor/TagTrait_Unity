1. AI 역할
- Unity 6 프로젝트 구조를 준비하는 Unity C# 개발자로 작업한다.
- 현재 단계에서는 TagTrait 시스템의 실제 태그 로직을 구현하지 않는다.
- Runtime Assembly와 EditMode Test Assembly의 참조 구조를 준비한다.
- Unity Test Framework용 환경 확인 테스트 코드를 작성한다.
- Unity Editor 내부 컴파일과 Test Runner 실행은 수행했다고 간주하지 않는다.
- 실제 컴파일과 테스트 실행은 사용자가 Unity에서 수행한다.

2. 확정 개발 환경
- Unity 버전: Unity 6.0 LTS (6000.0.68f1)
- API Compatibility Level: .NET Standard 2.1
- Unity Test Framework: 1.6.0
- Unity Test Framework는 이미 설치되어 있다.
- Packages/manifest.json과 Packages/packages-lock.json을 수정하지 않는다.
- 최소 지원 플랫폼: Windows PC
- 최종 확장 고려 플랫폼: 모바일
- 빌드 백엔드 고려 사항: IL2CPP

3. 현재 단계 목표
- TagTrait 시스템의 최소 폴더 구조를 준비한다.
- Runtime Assembly Definition을 생성한다.
- EditMode Test Assembly Definition을 생성하고 Runtime Assembly를 참조하도록 한다.
- 빈 EditMode 환경 테스트 코드를 작성한다.
- 기준 문서의 목표 배치 경로를 확인한다.
- 다음 개발 단계에서 Runtime 코드와 EditMode 테스트를 추가할 수 있는 상태를 만든다.

4. AI 작업 완료 기준
- 지정된 폴더와 파일이 생성 또는 확인되어 있다.
- Runtime Assembly Definition 설정이 본 프롬프트와 일치한다.
- EditMode Test Assembly Definition 설정이 본 프롬프트와 일치한다.
- 환경 확인용 EditMode 테스트 코드가 작성되어 있다.
- 실제 Unity 컴파일 및 테스트는 미실행 상태로 명확히 보고한다.
- 사용자가 Unity에서 확인해야 할 작업을 완료 보고에 명시한다.

5. 개발 단계 최종 완료 기준
- 사용자가 Unity Editor에서 프로젝트 컴파일 성공을 확인한다.
- 사용자가 Unity Test Runner의 EditMode에서 환경 확인 테스트 통과를 확인한다.

8.2. Assembly 설정
- Assembly 이름:
-- (TagTraitSystem.Runtime)
- Root Namespace:
-- (TagTraitSystem.Runtime)
- 다른 프로젝트 Assembly를 참조하지 않는다.
- Editor 전용으로 제한하지 않는다.
- 테스트 Assembly로 설정하지 않는다.
- Unsafe Code를 활성화하지 않는다.

- Version Defines를 추가하지 않는다.
- 플랫폼 제외 또는 포함 설정을 추가하지 않는다.

8.3. 책임

- Runtime Assembly는 Editor, Tests 또는 Samples Assembly를 참조해서는 안 된다.

10.5. 테스트 내용

- (Assert.Pass())를 사용한다.
- 실제 TagTrait 데이터 구조 또는 태그 로직을 구현하거나 참조하지 않는다.

11. 코딩 규칙
- private 필드는 별도 접두어 없이 camelCase를 사용한다.
- 중괄호는 BSD 스타일을 사용한다.
- (var)를 사용하지 않는다.
- XML 문서 주석은 영어로 작성한다.
- 일반 코드 주석은 한글로 작성한다.
- 테스트 메서드 이름은 영어로 작성한다.
- 기능 구간이 없으므로 환경 확인 테스트에 (#region)을 억지로 추가하지 않는다.
- Nullable Reference Type을 도입하지 않는다.

12. 기존 파일 처리
- 작업 전 (Assets/TagTraitSystem) 이하의 기존 폴더와 파일을 확인한다.
- 같은 목적의 Assembly Definition이나 테스트 파일이 이미 있으면 중복 생성하지 않는다.
- 기존 파일 설정이 본 프롬프트와 다르면 즉시 덮어쓰지 않는다.
- 차이점, 예상 영향 범위 및 권장 수정안을 보고한다.
- 관련 없는 기존 Assembly Definition, 테스트, 프로젝트 코드 또는 설정을 수정하지 않는다.
- 기존 공개 API는 현재 단계에서 존재하지 않는 것이 정상이다.

13. .meta 파일 처리
- (.meta) 파일을 수동으로 생성하지 않는다.
- 임의 GUID를 작성하지 않는다.
- Unity Editor가 필요한 (.meta) 파일을 생성하도록 둔다.
- 이미 존재하는 (.meta) 파일은 수정하거나 삭제하지 않는다.
- 파일 이동이 필요한 경우 기존 (.meta) 파일을 분리하거나 유실하지 않는다.
