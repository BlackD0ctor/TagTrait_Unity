- 기준 충돌 시 현재 단계 프롬프트, 공통 규칙, 원본 설계서, 이전 완료 기록, 현재 코드·테스트, 일반 관례 순으로 판단한다.
- 충돌이 발견되면 임의로 해결하지 말고 문제, 영향 범위 및 권장안을 보고한다.

1. 목표
- public bool PerishableTagAdd(TagDefinition definition, float duration, GameObject source = null)를 구현한다.
- duration 유효성 검사와 소수점 첫째 자리 정규화를 구현한다.
- 정규화된 값을 새 TagInstance의 Duration과 RemainingTime 초기값에 동일하게 저장한다.
- RemainingTime은 초기 저장 시에만 정규화값을 사용한다.
- 이후 Tick에서 감소한 RemainingTime은 매번 정규화하거나 반올림하지 않는다.
- 기존 Trait 생명주기, 이벤트, 동일 TagID 재진입 가드를 적용한다.
- Tick과 만료는 현재 단계에서 구현하지 않는다.

2. 이전 단계 전제
- TagContainer 조회, 영구 TagAdd/TagSub, 이벤트, Trait 생명주기, TagActivate와 재진입 가드가 사용자 Unity 검증을 통과한 것으로 전제한다.
- 전제가 충족되지 않으면 구현을 시작하지 말고 누락 내용을 보고한다.

3. 수정·생성 파일
- 수정: Assets/TagTraitSystem/Runtime/Components/TagContainer.cs
- 생성: Assets/TagTraitSystem/Tests/EditMode/TagContainerPerishableAddTests.cs
- TagInstance 생성자와 기존 데이터 구조는 변경하지 않는다.
- Tick 관련 API와 코드를 조기 추가하지 않는다.

4. Definition 검증
- TagAdd와 동일한 검증 규칙을 사용한다.
- 차단 조건:
-- Null Definition
-- TagID.None
-- Null 또는 빈 TagName
-- Keyword + Trait 존재
-- 비Keyword + Trait Null
- 허용 조건:
-- Keyword + Trait Null
-- Category.None + Trait 존재
-- 공백만 있는 이름
-- 앞뒤 공백 이름
- TagAdd와 PerishableTagAdd가 private 검증 helper를 공유하도록 리팩터링한다.
- 권장 시그니처: private bool ValidateDefinitionForAdd(TagDefinition definition)
- public/internal API로 노출하지 않는다.
- 기존 TagAdd의 반환값과 로그 정책을 바꾸지 않는다.

5. duration 유효성
- 유효 조건: 유한한 float이며 duration > 0f
- Error 로그 후 false:
-- 0f
-- 음수
-- float.NaN
-- float.PositiveInfinity
-- float.NegativeInfinity
- 실패 시 Instance 생성, 상태 변경, Trait 호출, 이벤트, 재진입 가드 획득이 없어야 한다.
- 로그는 영어로 작성한다.
- 직접 Debug.LogError 인접 위치에 단계 12에서 TagDiagnostics로 교체한다는 한글 주석을 둔다.

6. duration 정규화
- 태그 추가 시 한 번만 정규화한다.
- 소수점 첫째 자리까지 반올림한다.
- MidpointRounding.AwayFromZero를 사용한다.
- 정규화 후 최소 0.1f를 보장한다.
- 작은 양수는 실패시키지 않는다.
- 예:
-- 1.24f -> 1.2f
-- 1.25f -> 1.3f
-- 1.26f -> 1.3f
-- 0.001f -> 0.1f
-- 0.04f -> 0.1f
-- 0.05f -> 0.1f
-- 0.15f -> 0.2f
- 권장 helper: private static bool TryNormalizeDuration(float duration, out float normalizedDuration)
- TagInstance 생성자는 정규화하지 않고 전달값을 그대로 저장한다.

7. 초기 상태
- Definition = 전달된 Definition
- Duration = 정규화된 duration
- RemainingTime = 정규화된 duration
- StackCount = 1
- IsPerishable = true
- 정규화된 값은 시스템이 확정한 실제 기준 지속시간이다.
- RemainingTime은 초기 저장에만 정규화된 값을 사용한다.
- 이후 감소 결과는 일반 float로 연속적으로 저장하며 재정규화하지 않는다.

8. 처리 순서
- Definition 검증
- duration 검증 및 정규화
- 동일 TagID 재진입 가드 획득
- 소멸 TagInstance 생성
- TryAddInstance 호출
- 정상 비Keyword Trait이면 OnAdd
- OnTagAdded 발생
- finally에서 가드 해제
- true 반환

9. Trait과 이벤트
- 정상 비Keyword: 등록 후 OnAdd, 이후 OnTagAdded
- 정상 Keyword: OnAdd 없음, OnTagAdded는 발생
- OnAdd 예외: Error 로그, 상태 유지, Added 이벤트 계속, true 반환
- Added 이벤트:
-- Reason = Added
-- PreviousDuration = 0f
-- PreviousRemainingTime = 0f
-- PreviousStackCount = 0
-- Instance = 실제 소멸 TagInstance
-- Source = 전달된 source
- Removed/Updated 이벤트는 발생시키지 않는다.

10. 중복 임시 정책
- 같은 Definition의 기존 태그가 있으면 false이다.
- 기존 영구 태그 + PerishableTagAdd: false
- 기존 소멸 태그 + PerishableTagAdd: false
- 시간 갱신, 전환, 스택 증가는 하지 않는다.
- Trait 및 이벤트 없음, 로그 없음
- 같은 ID의 다른 Definition은 Error와 false
- 이후 수정 단계:
-- 단계 7: 영구·소멸 전환
-- 단계 8: Refresh
-- 단계 9: MaxDuration
-- 단계 11: 소멸 StackCount
- 코드 주석과 완료 보고에 임시 정책임을 명시한다.

11. 재진입
- processingTagIDs를 사용한다.
- Trait과 이벤트 종료까지 같은 ID 가드를 유지한다.
- 같은 ID Add/Sub/Activate/PerishableTagAdd 재진입은 false이다.
- 다른 ID 변경과 조회 API는 허용한다.
- finally에서 반드시 가드를 해제한다.

- Duration과 초기 RemainingTime에 같은 정규화값 저장
- 작은 양수 최소 0.1f

- 비Keyword + Trait Null

- deltaTime 코드를 조기 작성하지 않는다.
- 기존 TagAdd 정책을 변경하지 않는다.
- 중복 소멸 태그 시간을 갱신하지 않는다.
- 영구 태그를 소멸 태그로 변환하지 않는다.
- 미래 StackPolicy를 조기 구현하지 않는다.
- 미실행 테스트나 미확인 컴파일 성공을 보고하지 않는다.
