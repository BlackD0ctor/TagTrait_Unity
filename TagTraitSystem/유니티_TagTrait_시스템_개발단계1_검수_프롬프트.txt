4. Enum 검수

4.1. TagID
- public enum인지 확인한다.
- 다음 값이 정확한지 확인한다.
-- None = 0
-- TestAlpha = 1
-- TestBeta = 2
-- TestGamma = 3
- 테스트용 임시 ID 주석이 있는지 확인한다.
- 추가 실제 태그 ID가 임의로 포함되지 않았는지 확인한다.
- 중복 숫자값이 없는지 확인한다.

4.2. TagCategory
- 다음 값이 정확한지 확인한다.
-- None = 0
-- Keyword = 1
-- Status = 2
- 추가 카테고리가 임의로 포함되지 않았는지 확인한다.

4.3. StackPolicy
- 다음 값이 정확한지 확인한다.
-- None = 0
-- MaxDuration = 1
-- Refresh = 2
-- StackCount = 3
- Extend가 조기 추가되지 않았는지 확인한다.

4.4. TagChangeReason
- 다음 값이 정확한지 확인한다.
-- Added = 0
-- Removed = 1
-- DurationRefreshed = 2
-- DurationExtendedToMax = 3
-- StackIncreased = 4
-- StackDecreased = 5
-- ChangedToPermanent = 6
-- Expired = 7

4.5. ExclusiveGroup
- None = 0만 존재하는지 확인한다.
- enum 숫자값이 우선순위 의미로 사용되지 않았는지 확인한다.

5. TagTraitSO 검수

6. TagDefinition 검수

6.1. 클래스 및 CreateAssetMenu
- public sealed class인지 확인한다.
- ScriptableObject를 상속하는지 확인한다.
- CreateAssetMenu 값이 다음과 같은지 확인한다.
-- fileName = NewTagDefinition
-- menuName = TagTraitSystem/Tag Definition
-- order = 0

6.2. 필드
- 다음 private 직렬화 필드가 정확히 존재하는지 확인한다.
-- tagID
-- tagName
-- category
-- trait
-- stackPolicy
-- maxStackCount
-- exclusiveGroup
-- priority
-- isSaveable
- tagName 기본값이 string.Empty인지 확인한다.
- maxStackCount 기본값이 1인지 확인한다.
- 별도 접두어 없는 camelCase인지 확인한다.
- Runtime 상태 필드가 추가되지 않았는지 확인한다.
-- Duration
-- RemainingTime
-- StackCount
-- IsPerishable
- MaxStackCount를 TagInstance Runtime 필드로 복제하지 않았는지 확인한다.

6.3. 프로퍼티
- 다음 읽기 전용 프로퍼티가 정확히 존재하는지 확인한다.
-- TagID
-- TagName
-- Category
-- Trait
-- StackPolicy
-- MaxStackCount
-- ExclusiveGroup
-- Priority
-- IsSaveable
-- IsKeywordTag
- public setter가 없는지 확인한다.
- MaxStackCount가 maxStackCount를 그대로 반환하는지 확인한다.
- MaxStackCount를 오브젝트별 또는 Runtime 값으로 계산하지 않는지 확인한다.
- IsKeywordTag가 Category == TagCategory.Keyword만 확인하는지 검수한다.
- Trait Null 여부가 키워드 판정에 사용되지 않았는지 확인한다.

6.4. 범위 제한
- IsValid 또는 Validate가 추가되지 않았는지 확인한다.
- OnValidate가 구현되지 않았는지 확인한다.
- 중복 ID 검색이 구현되지 않았는지 확인한다.
- Keyword-Trait 조합 차단이 아직 Runtime 코드에 들어가지 않았는지 확인한다.
- 비Keyword Trait Null 차단이 아직 구현되지 않았는지 확인한다.
- MaxStackCount < 1 검증이나 자동 보정이 조기 구현되지 않았는지 확인한다.
- MaxStackCount == 1을 오류로 취급하지 않았는지 확인한다.
- 최대 스택 도달 처리와 오브젝트별 최대치 override가 조기 구현되지 않았는지 확인한다.

7. TagInstance 검수

7.1. 클래스
- public sealed class인지 확인한다.
- MonoBehaviour, ScriptableObject 또는 struct가 아닌 일반 참조형 클래스인지 확인한다.

7.2. 프로퍼티
- 다음 프로퍼티가 존재하는지 확인한다.
-- Definition
-- Duration
-- RemainingTime
-- StackCount
-- IsPerishable
- TagInstance에 MaxStackCount 프로퍼티나 별도 저장 필드가 없는지 확인한다.
- Definition은 getter only인지 확인한다.
- 나머지 상태 프로퍼티의 setter가 private인지 확인한다.
- public setter나 외부 상태 변경 API가 없는지 확인한다.

7.3. 영구 생성자
- internal TagInstance(TagDefinition definition)인지 확인한다.
- Null Definition에서 ArgumentNullException을 발생시키는지 확인한다.
- 정상 초기값이 다음과 같은지 확인한다.
-- Duration 0f
-- RemainingTime 0f
-- StackCount 1
-- IsPerishable false

7.4. 소멸 생성자
- internal TagInstance(TagDefinition definition, float duration)인지 확인한다.
- Null Definition에서 ArgumentNullException을 발생시키는지 확인한다.
- 전달한 duration을 그대로 Duration과 RemainingTime에 저장하는지 확인한다.
- StackCount 1인지 확인한다.
- IsPerishable true인지 확인한다.

7.5. 범위 제한
- 시간 반올림이나 최소값 보정이 조기 구현되지 않았는지 확인한다.
- NaN 또는 Infinity 검증이 조기 구현되지 않았는지 확인한다.
- 상태 변경 메서드가 조기 추가되지 않았는지 확인한다.
- TagContainer 또는 Dictionary 의존성이 없는지 확인한다.

17. 문제 분류 기준

17.1. 차단 문제
- enum 숫자값 불일치
- TagDefinition과 TagInstance 책임 혼합
- TagDefinition에 현재 StackCount 같은 런타임 상태 필드 추가
- MaxStackCount 누락, 기본값 불일치 또는 TagDefinition 외부에 원본 저장
- TagInstance에 MaxStackCount 복제 필드 또는 오브젝트별 override 추가
- TagInstance public setter 또는 public 생성자 노출
- IsKeywordTag가 Trait Null 기준으로 구현됨
- 시간 정규화나 TagContainer 기능 조기 구현
- Test Assembly가 internal 생성자에 접근할 수 없음
- 기존 기준 문서 또는 Package 파일 무단 변경
- 테스트 실행 결과 허위 보고

17.2. 수정 권장 문제
- 파일 경로 또는 네임스페이스 오타
- CreateAssetMenu 값 불일치
- MaxStackCount 전용 테스트 일부 누락
- XML 주석 또는 코드 스타일 위반
- 일부 기본값 테스트 누락
- 불필요한 using 또는 #region
- 완료 보고 항목 일부 누락

17.3. 사용자 확인 필요
- Unity 컴파일 결과
- EditMode 테스트 실행 결과
- CreateAssetMenu 표시 결과
- Inspector 직렬화 결과
- Assembly Definition 참조 및 InternalsVisibleTo 적용 결과
